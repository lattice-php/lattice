The date time input collects a calendar date and time in the active frontend timezone. Validation
returns a `CarbonImmutable`.

## Timezone behavior

The picker uses the frontend timezone configured through `lattice.i18n.timezone`. Submitted values
include that timezone, so validation returns a `CarbonImmutable` in the user's timezone by default.

```php
DateTimeInput::make('starts_at', 'Starts at');
```

To store the value in the app timezone, opt in:

```php
DateTimeInput::make('starts_at', 'Starts at')
    ->convertTimeZone();
```

To convert to a specific timezone, pass it explicitly:

```php
DateTimeInput::make('starts_at', 'Starts at')
    ->convertTimeZone('UTC');
```
