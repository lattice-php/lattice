A table is a listing backed by a **data source**. You declare columns in PHP; Lattice renders the
React table and handles sorting, filtering, and pagination, fetching rows from the table's [endpoint](/advanced/security/).
An Eloquent-backed source ships out of the box, and you can back a table with anything else by
implementing the source interface.

## Defining a table

Extend `TableDefinition` (or, for a database-backed table, `EloquentTableDefinition`) and declare the
columns. The `#[AsTable]` attribute gives the table a stable id so it can be discovered and addressed by
its own endpoint.

```php
use Illuminate\Database\Eloquent\Builder;
use Lattice\Lattice\Attributes\AsTable;
use Lattice\Lattice\Tables\Columns\NumberColumn;
use Lattice\Lattice\Tables\Columns\TextColumn;
use Lattice\Lattice\Tables\Sources\Eloquent\EloquentTableDefinition;
use Lattice\Lattice\Tables\TableQuery;

#[AsTable('app.products')]
class ProductsTable extends EloquentTableDefinition
{
    public function columns(): array
    {
        return [
            TextColumn::make('name')->sortable()->filterable(),
            NumberColumn::make('price')->sortable(),
        ];
    }

    public function builder(TableQuery $query): Builder
    {
        return Product::query();
    }
}
```

`TableDefinition` exposes a handful of hooks beyond `columns()`:

| Method               | Purpose                                                                           |
| -------------------- | --------------------------------------------------------------------------------- |
| `columns()`          | The columns to render — see [Columns](/tables/columns/).                          |
| `source()`           | The [data source](#data-sources). `EloquentTableDefinition` builds one for you.   |
| `perPage()`          | Rows per page (default `25`).                                                     |
| `pagination()`       | The [pagination mode](/tables/sorting-filtering-pagination/#pagination).          |
| `striped()`          | Alternate row shading.                                                            |
| `resizableColumns()` | Let users drag column borders to resize. Widths persist per table in the browser. |
| `resizeIndicator()`  | Show a drag indicator on column borders when resizing is enabled.                 |
| `emptyLabel()`       | Text shown when the table has no rows (default "No results").                     |
| `actionsLabel()`     | Accessible label for the row-actions column (default "Actions").                  |
| `actions($row)`      | Per-row [actions](/tables/actions/).                                              |
| `bulkActions()`      | [Bulk actions](/tables/actions/#bulk-actions) for selected rows.                  |

Each hook is a method you override on the definition. For example, to enable resizable columns with a
drag indicator:

```php
public function resizableColumns(): bool
{
    return true;
}

public function resizeIndicator(): bool
{
    return true;
}
```

Resized widths are remembered per table in the browser, with a control to reset them back to the
defaults.

## Data sources

A table's rows come from a `TableSource`. The interface has three methods — one to fetch a page, and
two that resolve rows for bulk actions:

```php
interface TableSource
{
    public function query(TableQuery $query): TableResult;
    public function resolveMatching(TableQuery $query): Collection;
    public function resolveSelection(array $keys): Collection;
}
```

- `TableQuery` is the read model of the current request — its filters, sorts, page, and page size.
- `TableResult` wraps the rows and pagination metadata. Build one from a paginator
  (`TableResult::fromPaginator()`, `fromSimplePaginator()`) or a plain collection
  (`TableResult::fromItems()`).

Most tables use the built-in [Eloquent source](/tables/eloquent-tables/). For an in-memory list, an
API, or a search index, implement `TableSource` yourself or use `CallbackTableSource`, which takes the
three methods as closures:

```php
public function source(): TableSource
{
    return new CallbackTableSource(
        fn (TableQuery $query) => TableResult::fromItems($this->rows($query)),
    );
}
```

## Rendering a table

Render a table on a page with `Table::use()`, passing the definition class:

```php
use Lattice\Lattice\Tables\Components\Table;

Table::use(ProductsTable::class);
```

`Table::lazy()` renders the same table but defers the first data load until the component mounts —
useful inside a tab or a section that isn't visible on page load.
